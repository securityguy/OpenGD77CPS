承認基準

音声またはデータをチャンネルで送信できるようにするタイミングを決定します。これは、すでに使用されているチャンネルで無線が送信されないようにするために使用されます。無線の送信周波数と受信周波数が異なる場合は、受信周波数のみのアクティビティが監視されます。受信周波数でアクティビティが見つからない場合、無線は、使用されている場合でも、送信周波数で送信することをユーザーに許可します。これはチャンネル全体の機能です。

Option
いつでもTプッシュツートーク（PTT）ボタンが押されると、無線は常に送信されます。このオプションは、「Impolite」チャンネルアクセスとも呼ばれます（Capacity PlusPersonalityおよびLinkedCapacity Plus Personalityチャンネルでは使用できません）。
Channel Free無線は、送信を許可する前にアイドルチャンネルをチェックします。このオプションは、「PolitetoAll」チャンネルアクセスとも呼ばれます。
Correct PL無線は、送信を許可する前にPLの一致をチェックします。このオプションは、Rx Squelch TypeがTPLまたはDPLに設定されている場合にのみ使用できます（アナログチャンネルの場合のみ）。
Color Code Free無線機は、送信を許可する前に、指定されたカラーコードが使用されていないかどうかを確認します（すでに進行中のグループ通話を除く）。このオプションは、「Polite to Own DigitalSystem」チャンネルアクセスとも呼ばれます（デジタルチャンネルのみ）。

Note
  • これらの設定は、すべての起動タイプで使用されるわけではありません。たとえば、緊急警報の声は常に失礼であり、データと制御メッセージは常に丁寧です。例外的なケースは、緊急警報の送信に、失礼なチャンネル訪問と礼儀チャンネルサービスの両方が含まれる場合です。受信機能のみを無効にする必要があります。
